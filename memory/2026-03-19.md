## 教训：修改配置文件的规范流程 (12:46)

### 问题
修改 `openclaw.json` 时操作不当，导致文件格式错误（JSON 重复内容），gateway 无法启动，用户需要手动覆盖配置文件恢复。

### 根因
1. 未备份就编辑关键配置文件
2. 编辑时未注意格式完整性
3. 未验证 JSON 格式就重启服务

### 正确流程（以后必须遵守）
1. **备份**: `cp openclaw.json openclaw.json.bak`
2. **编辑**: 小心修改，确保 JSON 结构完整
3. **验证**: `jq . openclaw.json > /dev/null && echo "格式正确"`
4. **重启**: 确认无误后再重启 gateway

### 重要原则
- 系统配置文件（JSON/YAML）不像普通文本，搞坏了会导致服务无法启动
- 多一行少一行都可能引发问题
- 必须养成备份+验证的习惯
